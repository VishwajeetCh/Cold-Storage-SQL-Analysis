# ❄️ Cold Storage Data Analysis using SQL

## 📌 Project Overview
This project analyzes **cold storage warehouse operations** using SQL queries.  
The objective is to extract valuable insights related to **storage utilization, product expiry risks, revenue generation, and storage demand patterns**.

The analysis helps improve **inventory management, storage planning, and financial tracking** for cold storage facilities.

---

## 🗄️ Database Used
**ColdStorageDB**

The database consists of the following tables:

- Farmers
- Products
- StorageUnits
- StorageEntries
- Payments

These tables help track product storage, farmer information, warehouse capacity, and payment transactions.

---

## 🎯 Project Objectives

- Analyze **storage unit capacity utilization**
- Detect **products at risk of exceeding safe storage limits**
- Identify **top farmers generating the highest revenue**
- Determine **most stored product categories**
- Track **monthly revenue trends**

---

## 🔎 Key SQL Insights

### 1️⃣ Capacity Utilization Analysis
Identify storage units that are heavily utilized.

**Purpose**
- Detect units exceeding **80% capacity**
- Optimize warehouse storage planning

---

### 2️⃣ Expiry Risk Detection
Find products stored longer than their maximum allowed storage period.

**Purpose**
- Prevent product spoilage
- Improve inventory management

---

### 3️⃣ Top Revenue Generating Farmers
Identify farmers contributing the most revenue to the cold storage facility.

**Purpose**
- Recognize high-value customers
- Improve farmer engagement strategies

---

### 4️⃣ Most Stored Product Category
Analyze which product categories occupy the highest storage volume.

**Purpose**
- Understand storage demand patterns
- Plan infrastructure based on product types

---

### 5️⃣ Monthly Revenue Analysis
Calculate revenue generated from farmer payments over time.

**Purpose**
- Track financial performance
- Identify revenue trends
